# Real peer data — provenance & sources

The executive-comp **peer-group screener** ([`peer_universe.csv`](../foundation/data/acme/peer_universe.csv)) benchmarks the synthetic subject (Acme) against **real public companies**. This file documents the provenance of every peer figure. It is an **illustrative, dated snapshot** for a methodology demo — **not** investment, accounting, or comp advice; verify against each company's SEC filings for current actuals. Figures were sourced by automated research against reputable public sources (company 10-K / IR / SEC EDGAR / stockanalysis.com) and are pending a human expert review.

- **As-of dates vary by row — the table is authoritative per figure, not this single date.** Most market caps are ~2026-07-02, but some are earlier (2026-05/06) where that was the best available point; revenue is each company's latest-reported FY **or** TTM and the fiscal period differs by company (calendar-year, Jan-31, Jun-30, etc.), so each cell carries its own period label. Treat any "as of ~2026-07-02" shorthand as approximate; the per-row labels govern. Tickers were validated as-of collection (e.g. ZoomInfo trades as **GTM**, formerly ZI).
- **Acme is not modeled on any company** — excluded or included. Its size, financials, and every ISS pay/TSR figure are synthetic constructions; Acme (ticker `ACMQ`) is not represented in this table.
- **Acme is a premium-multiple subject** by construction: ~7.5x revenue ($6.4B cap on $852M revenue) ≈ the ~P93 of this universe (peer median ~3x). Because the screen hard-gates on BOTH revenue and market cap around the subject, this premium tilts the selected group toward higher-multiple peers — itself a realistic comp-committee dynamic worth disclosing.
- **Excluded** (sourced but dropped): companies acquired / taken private since 2024 (Confluent→IBM, Squarespace, Smartsheet, Instructure, Couchbase, Clearwater, Semrush, Jamf, Olo) are removed because they no longer trade as independent public comparators.
- **GICS labels span several sectors — the gate is a software/SaaS GROUP, not one code.** GICS deliberately fragments SaaS: HCM software (Paycom, Paylocity) sits in **Industrials / "Human Resource & Employment Services"**, payments SaaS (Toast, Marqeta) in **Financials / "Transaction & Payment Processing Services"**. So the peer screener gates on membership in a documented **software/SaaS peer group** (the set of GICS sub-industries a committee treats as software peers), *not* an exact "Application Software" match — which mirrors real committee practice and does not depend on a firm's exact GICS Level-4 being publicly exposed. Sub-industries a software committee would NOT treat as software peers are OUT of the group and screened out as non-software: **"Interactive Media & Services"** (local ads/reviews — Yelp, ZoomInfo) and **"Health Care Technology"** (healthcare-data — Health Catalyst, Definitive Healthcare). Those four are size-excluded regardless, so the shipped peer set is unaffected — the group is just kept tight + accurate. Each row shows its **true** sub-industry at best-available confidence: rows marked `low(GICS)` (Klaviyo, Descartes, Zeta) are confirmed software/SaaS businesses whose exact GICS Level-4 is **not** publicly exposed (they are not S&P 500/400/600 constituents), so the Level-4 is best-evidence only; the group membership (software/SaaS) is solid regardless.
- **Ticker/name changes** are carried forward: BigCommerce → **Commerce.com, Inc.** (ticker `BIGC`→`CMRC`, Aug 1 2025); ZoomInfo → ticker `GTM` (formerly `ZI`).
- **ISS pay screen + rTSR valuation** run on a **separate synthetic** universe — no real name ever carries a fabricated pay/TSR figure.
